# Cal AI Clone — Design Spec
**Date:** 2026-04-02  
**Goal:** Learning / portfolio app — functional MVP cloning the core Cal AI calorie tracking experience  
**Platform:** Cross-platform iOS + Android via Expo (React Native)  
**Auth:** Local-only (no backend, no login — all data in on-device SQLite)

## 3. Data Model

### SQLite Tables (Drizzle schema)

```typescript
// profiles — one row, created at end of onboarding
profiles {
  id:                  integer primaryKey autoincrement
  goal:                text    // 'lose' | 'maintain' | 'gain'
  gender:              text    // 'male' | 'female' | 'other'
  birthday:            text    // ISO date string
  currentWeightKg:     real
  targetWeightKg:      real
  heightCm:            real
  activityLevel:       text    // 'sedentary' | 'light' | 'moderate' | 'active'
  dietaryPreferences:  text    // JSON array string
  dailyCalories:       integer
  dailyProteinG:       integer
  dailyCarbsG:         integer
  dailyFatG:           integer
  createdAt:           text    // ISO timestamp
}

// dailyLogs — one row per calendar day, created on first log of day
dailyLogs {
  id:       integer primaryKey autoincrement
  date:     text unique        // 'YYYY-MM-DD'
  waterMl:  integer default 0
}

// foodEntries — every logged food item
foodEntries {
  id:          integer primaryKey autoincrement
  dailyLogId:  integer references dailyLogs(id)
  name:        text
  calories:    integer
  proteinG:    real
  carbsG:      real
  fatG:        real
  servingSize: real
  servingUnit: text
  mealType:    text    // 'breakfast' | 'lunch' | 'dinner' | 'snack'
  source:      text    // 'ai_scan' | 'search' | 'manual'
  photoUri:    text    // nullable — local file path for AI-scanned items
  loggedAt:    text    // ISO timestamp
}

// exerciseEntries — logged workouts
exerciseEntries {
  id:               integer primaryKey autoincrement
  dailyLogId:       integer references dailyLogs(id)
  name:             text
  durationMinutes:  integer
  caloriesBurned:   integer
  loggedAt:         text
}

// weightLogs — manual weight entries for trend chart
weightLogs {
  id:        integer primaryKey autoincrement
  date:      text    // 'YYYY-MM-DD'
  weightKg:  real
  loggedAt:  text
}
```

### MMKV Keys

```
onboarding_complete   boolean   — gate for first-launch redirect
theme                 string    — 'light' | 'dark' | 'system'
weight_unit           string    — 'kg' | 'lbs'
gemini_api_key        string    — user-entered in Settings
nutritionix_app_id    string    — user-entered in Settings
nutritionix_api_key   string    — user-entered in Settings
```

### Zustand Stores

- **profileStore** — hydrated from `profiles` table on app start; holds goals + macro targets
- **dailyStore** — hydrated from today's `dailyLogs`, `foodEntries`, `exerciseEntries`; exposes `caloriesConsumed`, `caloriesRemaining`, `macrosConsumed`, `entries[]`, `waterMl`
- **settingsStore** — reads/writes MMKV directly; holds `theme`, `weightUnit`

## 4. Key Flows

### Food Photo Scan
1. Tap "+" FAB → bottom sheet: Camera / Search / Exercise / Water
2. `log/camera.tsx` — VisionCamera fullscreen; tap shutter
3. Encode photo to base64 → call `services/gemini.ts`
4. Gemini 2.0 Flash prompt:
   ```
   Analyze this food photo and return only valid JSON:
   {"name":"...","calories":0,"protein_g":0,"carbs_g":0,"fat_g":0,
    "serving_size":"...","health_score":0}
   ```
5. `log/scan-result.tsx` — full-bleed photo header, editable fields, health score bar
6. "Fix Results" → re-call Gemini with user's correction hint appended to prompt
7. "Done" → `INSERT INTO foodEntries`, `UPSERT dailyLogs` → update dailyStore → navigate back

### Onboarding → Plan Generation
1. 10-step stack (`goal` → `gender` → `birthday` → `current-weight` → `height` → `target-weight` → `activity` → `diet` → `results` → `plan`)
2. Each step writes to a local `onboardingState` Zustand slice (not persisted until final step)
3. `plan.tsx`: calculate TDEE via Mifflin-St Jeor formula in `utils/tdee.ts`; display as 4 editable donut rings (Calories, Protein, Carbs, Fat)
4. "Let's get started!" → `INSERT INTO profiles` → MMKV `onboarding_complete = true` → replace navigation to `/(tabs)/`

### Dashboard (Home)
- On screen focus: Drizzle query for today's `dailyLog` + `foodEntries` + `exerciseEntries`
- Net calories = consumed − exercise burned
- Calorie ring: remaining / goal; 3 macro pills
- Week strip: 7 days, active day highlighted; tap to view past day
- Streak: count consecutive days with ≥1 `foodEntry`
- Empty state: "Start tracking today's meals by taking a quick picture"

### Analytics
- Weight trend: `weightLogs` last 90 days → Gifted Charts line chart
- Weekly macros: `foodEntries` grouped by weekday → stacked bar chart
- BMI: `(latestWeight / height²) * 10000`
- Streak count display

### Dark Mode
- `settingsStore.theme` → written to MMKV
- `app/_layout.tsx` wraps app in NativeWind `colorScheme` provider
- All screens use `dark:` Tailwind variants for automatic switching

## 6. API Integration

### Gemini 2.0 Flash Vision
- **Endpoint:** `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent`
- **Auth:** `?key=GEMINI_API_KEY` (from MMKV)
- **Input:** base64 image + structured JSON prompt
- **Output:** JSON parsed from `candidates[0].content.parts[0].text`
- **Error handling:** If JSON parse fails or fields missing → show "Fix Results" prompt pre-filled

### Nutritionix Natural Language
- **Endpoint:** `https://trackapi.nutritionix.com/v2/natural/nutrients`
- **Auth:** `x-app-id` + `x-app-key` headers (from MMKV)
- **Input:** `{ query: "2 eggs and toast" }`
- **Output:** `foods[]` array with `nf_calories`, `nf_protein`, `nf_total_carbohydrate`, `nf_total_fat`

Both API keys are entered by the user once in Settings → stored in MMKV → read by `services/*.ts` at call time.
